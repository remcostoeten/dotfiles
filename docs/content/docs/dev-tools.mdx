## Overview

The dotfiles collection includes a comprehensive set of development tools designed to streamline common tasks and improve productivity. Each tool is cross-platform compatible and follows consistent interfaces.

## Available Tools

### File Operations

#### `copy` - Advanced File Copying

An intelligent file copying utility with conflict resolution and advanced options.

```bash
# Basic usage
copy source.txt destination.txt

# Copy with interactive conflict resolution
copy --interactive src/ dest/

# Copy with backup creation
copy --backup file.txt /path/to/destination/

# Show help
copy --help
```

**Features:**
- Interactive conflict resolution
- Automatic backup creation
- Progress indicators for large files
- Preserves file metadata
- Cross-platform compatibility

#### `file-utils` - File Management Operations

Utilities for common file management tasks.

```bash
# Find and organize files by type
file-utils organize --by-extension ./downloads

# Clean up temporary files
file-utils cleanup --temp-files

# Batch rename files
file-utils rename --pattern "*.txt" --replace "old" "new"
```

### Git Integration

#### `git-commit` - Interactive Git Commit Helper

Streamlined git commit workflow with conventional commit support.

```bash
# Interactive commit with conventional commits
git-commit

# Quick commit with message
git-commit -m "feat: add new feature"

# Commit with co-authors
git-commit --co-author "John Doe <john@example.com>"

# Show available commit types
git-commit --types
```

**Features:**
- Conventional commit format support
- Interactive commit type selection
- Co-author support
- Automatic staging options
- Commit message validation

**Supported Commit Types:**
- `feat`: New features
- `fix`: Bug fixes
- `docs`: Documentation changes
- `style`: Code style changes
- `refactor`: Code refactoring
- `test`: Test additions/modifications
- `chore`: Maintenance tasks

### Node.js Development

#### `node-clean` - Node.js Cache Cleaner

Clean Node.js caches and temporary files to resolve development issues.

```bash
# Clean npm cache
node-clean --cache

# Clean node_modules and package-lock.json
node-clean --modules

# Full clean (cache + modules + lock files)
node-clean --all

# Clean specific project
node-clean --project /path/to/project
```

#### `node-reinstall` - Complete Node.js Reinstallation

Completely reinstall Node.js dependencies with clean state.

```bash
# Reinstall in current directory
node-reinstall

# Reinstall with specific package manager
node-reinstall --manager pnpm

# Reinstall with cache clearing
node-reinstall --clean-cache

# Show what would be done (dry run)
node-reinstall --dry-run
```

#### `node-clean-dev` - Development Environment Cleaner

Clean development environment for Node.js projects.

```bash
# Clean development dependencies and cache
node-clean-dev

# Clean and rebuild development environment
node-clean-dev --rebuild

# Clean specific directories
node-clean-dev --target dist,build,.cache
```

### System Management

#### `kill-ports` - Port Management Utility

Kill processes running on specific ports to free them for development.

```bash
# Kill process on single port
kill-ports 3000

# Kill processes on multiple ports
kill-ports 3000,8080,9000

# Kill common development ports
kill-ports --dev-stack

# List processes before killing (interactive)
kill-ports --interactive 3000

# Show help with common port numbers
kill-ports --help
```

**Common Development Ports:**
- `3000` - React/Next.js development server
- `8080` - General development server
- `9000` - Webpack dev server
- `5173` - Vite development server
- `4200` - Angular development server

#### `webcam-mic` - Media Device Toggle

Toggle webcam and microphone settings for privacy and development.

```bash
# Toggle webcam
webcam-mic --webcam toggle

# Toggle microphone
webcam-mic --mic toggle

# Show current status
webcam-mic --status

# Disable both devices
webcam-mic --disable-all

# Enable both devices
webcam-mic --enable-all
```

### Code Quality

#### `remove-comments` - Source Code Comment Remover

Remove comments from source code files while preserving functionality.

```bash
# Remove comments from single file
remove-comments file.js

# Remove comments from multiple files
remove-comments src/**/*.js

# Remove comments with backup
remove-comments --backup src/

# Show supported file types
remove-comments --supported-types
```

**Supported Languages:**
- JavaScript/TypeScript (`.js`, `.ts`, `.jsx`, `.tsx`)
- Python (`.py`)
- CSS/SCSS (`.css`, `.scss`)
- HTML (`.html`)
- Shell scripts (`.sh`, `.fish`)

## Tool Configuration

### Environment Variables

Tools can be configured using environment variables:

```bash
# Set default package manager for Node.js tools
export DOTFILES_PACKAGE_MANAGER="pnpm"

# Set default git commit format
export DOTFILES_COMMIT_FORMAT="conventional"

# Set backup directory for file operations
export DOTFILES_BACKUP_DIR="$HOME/.dotfiles-backups"

# Enable verbose output for all tools
export DOTFILES_VERBOSE=true
```

### Configuration Files

Some tools support configuration files in your home directory:

```bash
# ~/.dotfiles-config.json
{
  "git": {
    "defaultCommitType": "feat",
    "requireDescription": true,
    "maxMessageLength": 72
  },
  "node": {
    "defaultPackageManager": "pnpm",
    "cleanCacheOnReinstall": true
  },
  "file": {
    "defaultBackupLocation": "~/.dotfiles-backups",
    "confirmBeforeOverwrite": true
  }
}
```

## Advanced Usage

### Chaining Commands

Tools can be chained together for complex workflows:

```bash
# Clean and reinstall Node.js project
node-clean --all && node-reinstall

# Kill development ports and restart
kill-ports --dev-stack && npm run dev

# Clean git working directory and commit
git clean -fd && git-commit
```

### Scripting Integration

Use tools in your own scripts:

```bash
#!/bin/bash
# deployment-prep.sh

echo "Preparing for deployment..."

# Clean Node.js environment
node-clean --all

# Reinstall dependencies
node-reinstall --manager pnpm

# Run tests
npm test

# Create deployment commit
git-commit -m "chore: prepare for deployment"

echo "Ready for deployment!"
```

### IDE Integration

#### VS Code Tasks

Add to `.vscode/tasks.json`:

```json
{
  "version": "2.0.0",
  "tasks": [
    {
      "label": "Clean Node Environment",
      "type": "shell",
      "command": "node-clean",
      "args": ["--all"],
      "group": "build"
    },
    {
      "label": "Kill Dev Ports",
      "type": "shell", 
      "command": "kill-ports",
      "args": ["--dev-stack"],
      "group": "build"
    }
  ]
}
```

#### Shell Aliases

Add to your shell configuration:

```fish
# Fish shell aliases
alias nc="node-clean"
alias nr="node-reinstall"
alias kp="kill-ports"
alias gc="git-commit"
```

## Performance and Monitoring

### Performance Tips

1. **Use specific targeting**: Only clean/process what you need
2. **Leverage dry-run modes**: Test commands before execution
3. **Configure defaults**: Set environment variables for common options
4. **Use interactive modes**: Avoid accidental data loss

### Monitoring Usage

Track tool usage with built-in logging:

```bash
# Enable logging for all tools
export DOTFILES_LOG_USAGE=true
export DOTFILES_LOG_FILE="$HOME/.dotfiles-usage.log"

# View usage statistics
tail -f ~/.dotfiles-usage.log
```

## Getting Help

### Built-in Help

Every tool includes comprehensive help:

```bash
# Get help for any tool
<tool-name> --help
<tool-name> -h

# Get version information
<tool-name> --version
```

### Common Options

All tools support these common options:

- `--help`, `-h`: Show help information
- `--version`, `-v`: Show version
- `--verbose`: Enable verbose output
- `--dry-run`: Show what would be done without executing
- `--config`: Specify custom configuration file
