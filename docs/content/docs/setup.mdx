## Installation

### Prerequisites

Before installing, ensure you have:

- Git installed on your system
- Shell access (bash, zsh, or fish)
- Basic command line familiarity

### Quick Installation

The fastest way to get started:

```bash
# Clone the repository to the standard config location
git clone https://github.com/remco-stoeten/dotfiles.git ~/.config/dotfiles

# Navigate to the directory
cd ~/.config/dotfiles

# Make all scripts executable
chmod +x bin/* scripts/**/*.sh internal/**/*.fish env/**/*.sh
```

### Platform-Specific Setup

#### Linux (Ubuntu/Debian)

```bash
# Initialize Linux environment
source ~/.config/dotfiles/env/loader.sh

# Install Fish shell (if not already installed)
sudo apt update && sudo apt install fish

# Set Fish as default shell
chsh -s $(which fish)

# Load Fish configuration
source ~/.config/dotfiles/env/init.fish
```

#### MacOS

```bash
# Initialize macOS environment
source ~/.config/dotfiles/env/loader.sh

# Install Fish shell via Homebrew
brew install fish

# Add Fish to shells
echo $(which fish) | sudo tee -a /etc/shells

# Set Fish as default shell
chsh -s $(which fish)

# Load Fish configuration
source ~/.config/dotfiles/env/init.fish
```

## Configuration

### Environment Variables

The dotfiles system uses modular environment management:

```bash
# View current environment setup
cat ~/.config/dotfiles/env/$(uname -s | tr '[:upper:]' '[:lower:]')/env.sh

# Available modules are listed in:
cat ~/.config/dotfiles/env/$(uname -s | tr '[:upper:]' '[:lower:]')/modules.list
```

### Fish Shell Configuration

After installation, Fish shell will have access to:

```fish
# Custom functions
copy             # Advanced file copying
git-commit       # Interactive git commit helper
kill-ports       # Kill processes on specific ports
node-clean       # Clean Node.js cache and dependencies
webcam-mic       # Toggle webcam and microphone

# Node.js development
node-reinstall   # Completely reinstall Node dependencies
node-clean-dev   # Clean development environment
```

### Adding to PATH

The setup automatically adds the `bin/` directory to your PATH. To verify:

```bash
echo $PATH | grep -o '\.config/dotfiles/bin'
```

## Verification

### Test Installation

Run these commands to verify everything is working:

```bash
# Test basic utilities
copy --help
kill-ports --help

# Test Fish functions (if using Fish shell)
fish -c "functions | grep -E '(copy|git-commit|kill-ports)'"

# Test environment loading
source ~/.config/dotfiles/env/loader.sh && echo "Environment loaded successfully"
```

### Available Commands

After setup, you'll have access to these utilities:

| Command | Description |
|---------|-------------|
| `copy` | Advanced file copying with conflict resolution |
| `git-commit` | Interactive git commit with conventional commits |
| `kill-ports` | Kill processes running on specific ports |
| `node-clean` | Clean Node.js dependencies and cache |
| `node-reinstall` | Completely reinstall Node dependencies |
| `webcam-mic` | Toggle webcam and microphone settings |
| `remove-comments` | Remove comments from source files |

## Customization

### Adding Custom Scripts

Create new utilities in the `bin/` directory:

```bash
# Create a new utility
touch ~/.config/dotfiles/bin/my-tool
chmod +x ~/.config/dotfiles/bin/my-tool

# Edit with your preferred editor
$EDITOR ~/.config/dotfiles/bin/my-tool
```

### Modifying Environment

Edit platform-specific environment files:

```bash
# Linux environment
$EDITOR ~/.config/dotfiles/env/linux/env.sh

# macOS environment  
$EDITOR ~/.config/dotfiles/env/macos/env.sh

# Common environment (shared across platforms)
$EDITOR ~/.config/dotfiles/env/common/env.sh
```

### Fish Shell Customization

Add custom Fish functions:

```bash
# Create new function
$EDITOR ~/.config/dotfiles/internal/helpers/my-function-helper.fish

# Then create the main script
$EDITOR ~/.config/dotfiles/bin/my-function.fish
```

## Troubleshooting

### Common Issues

#### Permission Denied
```bash
# Fix permissions
chmod +x ~/.config/dotfiles/bin/*
chmod +x ~/.config/dotfiles/scripts/**/*.sh
```

#### Command Not Found
```bash
# Verify PATH
echo $PATH | grep dotfiles

# Reload environment
source ~/.config/dotfiles/env/loader.sh
```

#### Fish Functions Not Available
```bash
# Reload Fish configuration
source ~/.config/dotfiles/env/init.fish

# Check function loading
fish -c "functions | head -10"
```

### Getting Help

Each utility includes help documentation:

```bash
# Get help for any command
copy --help
git-commit --help
kill-ports --help
```
