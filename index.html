Technical Architecture
Design
Tech Stack
Primary Languages: TypeScript, Bash, Fish Shell, Python
UI Framework: React with @opentui/react (TUI framework)
Runtime: Bun for TypeScript/JavaScript execution
Package Management: APT, Snap, NPM, GitHub Releases, CURL downloads
Shell: Fish (with extensive custom functions and aliases)
Editor: Neovim with LazyVim configuration
Directory Architecture
Separation of Concerns:
‚Ä¢ `bin/` - Executables on PATH (wrappers, direct scripts)
‚Ä¢ `scripts/` - Implementation logic (called by bin/)
‚Ä¢ `configs/` - Application configurations (Fish, Neovim, Kitty, etc.)
‚Ä¢ `opentui-setup/` - Interactive setup system (TypeScript/React)
‚Ä¢ `shell-setup/` - Modular shell installer (package groups)

Why this structure?
‚Ä¢ Clean separation between interface (bin) and logic (scripts)
‚Ä¢ Allows updating implementations without changing PATH
‚Ä¢ Modular: each script can be used independently
‚Ä¢ Version-controlled configs with atomic symlinks
Setup System Architecture
The setup system is built in layers:

Layer 1: Entry Points
‚Ä¢ `install.sh` - Bootstrap on fresh system (installs Git + Bun)
‚Ä¢ `interactive-setup.ts` - Main interactive TUI (`bun run interactive`)
‚Ä¢ `cli-setup.ts` - Headless/non-interactive (`bun run cli`)
‚Ä¢ `cli-manager.ts` - Individual package management (`bun run manager`)

Layer 2: Core Modules
‚Ä¢ `packages.ts` - Package definitions (18 categories, 100+ packages)
‚Ä¢ `executor.ts` - Command execution with retry logic
‚Ä¢ `install-manager.ts` - Advanced installer (retry, rollback, progress)

Layer 3: Configuration
‚Ä¢ `dotfiles-config.ts` / `dotfiles-setup.ts` - Symlink management
‚Ä¢ `system-config.ts` - GNOME, browser, wallpaper, sudo config
‚Ä¢ `scripts-executable.ts` - Make all scripts executable
‚Ä¢ `nerd-fonts.ts` - Font installation
‚Ä¢ `gh-extensions.ts` - GitHub CLI extensions

Layer 4: Verification
‚Ä¢ `verification.ts` - Post-install checks and validation
Error Handling Strategy
Multi-Level Failure Recovery:
1. Auto-retry: Up to 3 attempts with exponential backoff
2. Dependency resolution: Auto-install missing dependencies (e.g., snapd)
3. Graceful degradation: Continue on non-critical failures
4. Rollback: Attempt to remove partially installed packages
5. Progress tracking: Save state to `~/.dotfiles/setup/progress.json`
6. Resume capability: Can continue from last successful step
Package Installation Flow
For each package:
1. Check if already installed (version detection)
2. Determine installation method (APT/Snap/NPM/GitHub/CURL)
3. Install dependencies if needed
4. Execute installation with timeout and error handling
5. Verify installation success
6. On failure: retry with backoff, fallback to alternative method
7. Update progress tracker
8. Run post-install hooks (if defined)
Configuration Management
Symlink Strategy:
‚Ä¢ All configs symlinked to dotfiles repo (single source of truth)
‚Ä¢ Atomic operations: backup existing ‚Üí create symlink
‚Ä¢ Verification: check symlink targets before and after
‚Ä¢ Rollback: restore from backup if symlink fails

Files managed:
‚Ä¢ Fish: `~/.config/fish/config.fish` ‚Üí `dotfiles/cfg`
‚Ä¢ Neovim: `~/.config/nvim` ‚Üí `dotfiles/configs/nvim`
‚Ä¢ Kitty: `~/.config/kitty` ‚Üí `dotfiles/configs/kitty`
‚Ä¢ Git: `~/.gitconfig` ‚Üí `dotfiles/configs/git/.gitconfig`
‚Ä¢ WezTerm: `~/.config/wezterm` ‚Üí `dotfiles/configs/wezterm`
Fish Shell Architecture
Modular organization:
‚Ä¢ `core/` - Core setup (colors, env, init)
‚Ä¢ `aliases/` - Categorized aliases (dev, git, system, docker, etc.)
‚Ä¢ `functions/` - Custom Fish functions

Main entry: `cfg` file sources everything in correct order
Total size: 361 lines of aliases + custom functions
Features: Auto-completion, syntax highlighting, git integration
Performance & Optimization
‚Ä¢ Parallel package installations where possible
‚Ä¢ Lazy loading of shell functions and aliases
‚Ä¢ Cache package availability checks
‚Ä¢ Skip already-installed packages (version-aware)
‚Ä¢ Minimal shell startup time (Fish config optimized)
‚Ä¢ Bun for fast TypeScript execution

Wrapper Pattern: `bin/script-name` ‚Üí `scripts/script-name` (implementation)
